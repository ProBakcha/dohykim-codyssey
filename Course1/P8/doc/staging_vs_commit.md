# staging_vs_commit.md

## 차이 요약

- **git add**  
  → 변경 파일을 *스테이징 영역*에 추가 (커밋 준비 단계)

- **git commit**  
  → 스테이징 영역의 변경 내용을 *로컬 저장소*에 기록

## 핵심 비교

| 구분    | git add                  | git commit                         |
|---------|--------------------------|-------------------------------------|
| 역할    | 커밋할 파일 선택         | 선택된 파일을 저장소에 기록        |
| 시점    | 커밋 전                  | 커밋 시점                           |
| 취소    | 가능 (`git reset`)       | 복잡 (`git revert`, `reset`)       |
| 저장소 기록 | ❌                    | ✅                                  |
